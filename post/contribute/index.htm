Contributing to P

P is a young open source project with many areas for improvement. Pull requests are welcome at GitHub.

Building from source

npm install -g grunt-cli; # Installs CLI for Grunt, the build engine for P
npm install -g bower; # Installs Bower, a package manager used by P
git clone git://github.com/oztu/p.git; # Clone the P repository
cd p; # Enter into the P directory
npm install; # Install dev dependencies
bower install; # Installs runtime dependencies
grunt dev; # Start the dev builder

If all goes well, Grunt is now watching source files for changes and running a dev server on localhost:20501 where you can try out examples. Whenever you modify a file, grunt will rebuild the project automatically.

What's in the repo

  • build/ – Temporary build files, the dev server files are also put here
  • components/ – Managed by Bower, a package manager
  • dist/ – Distributable files
  • examples/
  • lib/ – Source files of the modules for the P project
    • Connection.js – Base connection class for nodes
    • ConnectionManager.js – The shared list of peers across node
    • P.js – The root of the project, exported as a global variable
    • JSONProtocol.js – The P protocol definition and helpers
    • WebRtcConnection.js – WebRTC-specific Connection
    • WebSocketConnection.js – WebSocket-specific Connection
  • node_modules/ – Managed by NPM, a package manager
  • bower.json – Bower component definition file
  • Gruntfile.js – Grunt task file, used to define build process
  • package.json – NPM module definition file

Creating a Release Build

Execute grunt release to generate a new p.js file in the dist/ folder. This is a packaged and minified version of P.
